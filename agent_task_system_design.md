## 一、设计目标

### 1.1 核心定位

本游戏本质是**以游戏为壳的 Agent 能力评测平台**。选手通过编写 Agent 参与游戏，在游戏的自然流程中接受多维度的 Agent 能力考察。游戏体验与评测目标高度耦合——选手在"玩游戏"的过程中，不知不觉被全方位评测。

### 1.2 考察维度全景

| 维度 | 核心考察点 | 当前状态 |
|------|-----------|---------|
| **A. 长流程任务** | 多步子任务分解、跨回合协调、依赖管理、失败恢复 | ✅ 已有 |
| **B. 工具使用** | 截图理解、工具选择（OCR/VLM）、网页交互决策 | ❌ 缺失 → **新增** |
| **C. 因果推理与博弈** | 情报推演、噪声过滤、双边谈判、信息差利用 | ❌ 缺失 → **新增** |
| **D. 跨回合记忆整合** | 长上下文管理、碎片拼凑、跨领域推理 | ❌ 缺失 → **新增** |
| **E. 多智能体协调** | 采集英雄+开拓者跨角色配合 | ✅ 已有，可强化 |

## 二、统一任务框架

### 2.1 触发方式

四类任务通过不同的方式触发，但共享同一个原则——**判题器返回自然语言，Agent 自主解读并行动**。

| 任务 | 触发源 | 触发方式 |
|------|------|---------|
| A-长流程任务 | 晨间新闻 | 新闻提及模糊地点 → 开拓者移动到目标区域 → 使用[探索] → 判题器返回 NL 任务详情 |
| B-网页任务 | 晨间新闻【世界事件】 | 新闻提及废弃据点有古代终端 → 开拓者移动到废弃据点 → 使用[探索] → 判题器返回 URL 和 token |
| C-推理任务 | 晨间新闻【世界事件】 | 每天自动推送情报，Agent 读取后推理 → 贸易站双边谈判时利用推理结论 |
| D-长上下文任务 | 晨间新闻【各地见闻】 | 每天自动推送碎片，Agent 存入记忆 → 集齐后自行推理并到目标地点执行 |

### 2.2 探索协议

开拓者使用技能 `explore`，判题器根据英雄当前坐标和游戏状态返回自然语言描述。对于网页任务，判题器额外返回终端截图路径。

```
// 选手 Agent 发送
{
  "hero_id": "pioneer_1",
  "skill": "explore"
}

// 判题器返回（示例：在废弃据点发现了古代交易终端）
{
  "result": "你在废弃据点的废墟中发现了一台仍能运转的古代交易终端。
             屏幕亮着微弱的蓝光，上面显示着装备采购界面。",
  "screenshot": "/session/shop/sess_001/screenshot_001.png"
}
```

判题器不告诉 Agent "这是网页任务"——判题器只返回一段文字加一张截图。Agent 拿到截图后，自己判断这是什么、该怎么处理。

### 2.3 网页任务的触发与交互流程

全程复用游戏已有技能 `explore` 和 `use`。

```
晨间新闻【世界事件】：
  "勘探队在废弃据点（坐标 18,6）发现一台仍能运转的古代交易终端。
   与之交互可能获得意想不到的奖励。"
    ↓
Agent 派开拓者移动到 (18,6)
    ↓
开拓者使用技能 explore
    ↓
判题器内部启动 Playwright → 渲染商店首页 → 截图 → 保存到文件
    ↓
返回: { result: "描述...", screenshot: "/session/.../xxx.png" }
    ↓
Agent 读取截图（OCR/VLM）→ 分析 → 决定下一个操作
    ↓
开拓者使用技能 use 发送操作指令
    ↓
判题器 Playwright 执行 → 重新截图 → 返回新截图
    ↓
循环直至 10 分钟窗口结束或 Agent 完成购买
```

### 2.4 use 技能——网页终端操作

Agent 通过开拓者的 `use` 技能发送对终端的操作。仅两种动作：

```
// 点击
{
  "hero_id": "pioneer_1",
  "skill": "use",
  "action": "click",
  "coordinate": {"x": 450, "y": 220}
}

// 滚动
{
  "hero_id": "pioneer_1",
  "skill": "use",
  "action": "scroll",
  "direction": "down",
  "length": 300
}

// 判题器每次返回
{
  "result": "ok",
  "screenshot": "/session/shop/sess_001/screenshot_003.png"
}
```

**交互示例**：

```
截图#1: Agent看到商品列表
  → use, scroll, down, 400          // 往下翻看
  → use, scroll, up, 400            // 翻回去，看清楚整个界面
  → use, click, {450, 180}          // 勾选商品#1
  → 截图#3: 复选框已勾选 ✓
  → use, click, {600, 700}          // 点击[购买]
  → 截图#4: 弹框"总价超过100金币" + [确定]
  → use, click, {400, 350}          // 点击[确定]关闭弹框
  → 截图#5: 回到购买页面，重新调整...
```

网页交互与游戏回合并行，Agent 自由决定何时结束操作。判题器以最终有效购买结果结算。

### 2.5 任务生命周期

```
[触发] → [Agent解读判题器返回] → [自主规划] → [执行]
                                                ├── 成功 → [游戏内发放奖励]
                                                ├── 失败 → Agent自行判断是否重试
                                                └── 超时 → 任务关闭
```

## 三、任务类型详设

---

### 【类型 A】长流程任务— 多步规划与执行

> **考察维度**：长任务分解、NL理解、多步执行、多Agent协调、失败恢复
> **状态**：✅ 已有，此处整合优化

#### 3A.1 子类型总览

| 子类型 | 简述 | 核心 Agent 能力 |
|--------|------|:----------|
| A1-瘟疫治疗 | 配方获取→材料采集→合成→指定地点使用 | 模糊寻路、跨英雄协调 |
| A2-石碑祭品 | NL谜语→物品映射→顺序推理→依次使用 | 谜语解读、顺序推理 |
| A3-残卷解谜 | 3卷散落→逐卷探索→答题/满足诉求 | 谜题解答、需求满足 |
| A4-流浪商人 | 追踪商人→NL描述→物品映射→以物易物 | 追踪规划、NL物品映射 |

#### 3A.2 任务示例：瘟疫治疗（A1）

```
世界新闻：
	城邦居民得了瘟疫，需要使用“特效药”治疗，配方随机出现在西方森林处，将“特效药”倒入湖泊中缓解瘟疫
Agent执行流程：
  1. 开拓者模糊寻路至"西方森林"区域
  2. 开拓者使用[探索] → 获知配方：
  3. 开拓者使用[探索] → 获知配方合成方式，比如（小麦 + 草药 + 淡水）
  4. Agent判断：需要采集英雄去收集小麦（农田）、草药（密林）、淡水（湖泊）
  5. 采集英雄依次收集材料 → 工坊合成"特效药"
  6. 开拓者携带特效药 → 模糊寻路至"湖泊"
  7. 开拓者使用[使用] → 将特效药倒入湖泊
  8. 任务完成 → 奖励图纸+积分
```

#### 3A.3 任务示例：石碑祭品（A2）

```
世界新闻：
	西边森林处出现了远古石碑，据说在石碑处以正确顺序放置祭品会有奖励，若错乱则石碑会吞噬祭品，甚至石碑破碎，需要一段时间恢复（5回合石碑冷却）

Agent执行流程：
	1. 开拓者模糊寻路至"西方森林"区域
	2. 开拓者使用[探索] → 获知一段话：
      “石碑上刻着四句谜语：
        1. "我来自熔炉之中，比石坚硬"
        2. "我来自年轮之中，撑起屋檐"
        3. "我来自河流之中，盛于陶罐"
        4. "我来自火山之中，遇火即燃"”
    3. Agent需要推理：
        1 → 铁矿石（熔炉、坚硬）
        2 → 木材（年轮、屋檐=房屋）
        3 → 淡水（河流、陶罐=盛水容器）
        4 → 硫磺（火山、燃烧）
	4. 返回城邦，取出需要的物品，若缺失需要采集英雄进行配合
	5. 返回石碑处，顺序放置物品（需要依次讲铁矿石、木材、淡水、硫磺在石碑处使用）
```

#### 3A.4 任务示例：残卷解谜（A3）

```
世界新闻：
    地图上散落了上古卷轴残卷，共3卷，分布在了东边山脉。
    找到并正确回答卷轴的问题，或满足卷轴的诉求，则会有奖励。
    若回答错误，卷轴会暂时失效，需要一段时间恢复（5回合冷却）。

Agent执行流程：
    1. 开拓者模糊寻路至"东边山脉"区域
    2. 开拓者使用[探索] → 判题器返回当前区域是否含有残卷
       若含有残卷，则返回该卷轴的问题/诉求，例如：
         - 答题型："越黑越亮，遇光则亡——我是什么？（答案2字）" → 星星
         - 诉求型："汝需使用1块铁石之力唤醒吾之睡眠" → 使用铁矿石×1
    3. Agent需要推理：
         - 答题型：理解谜面 → 推理答案 → 输入答案
         - 诉求型：NL描述 → 物品映射 → 确认背包/仓库是否有相应物品
    4. 若答案正确/物品正确 → 该残卷完成
       若错误 → 卷轴冷却5回合后重新开放
    5. 循环步骤 2-4，直到 3 卷残卷全部完成
    6. 任务完成 → 奖励图纸+积分
```

#### 3A.5 奖励

| 子类型 | 积分 | 图纸/物品 | 其他 |
|--------|:---:|------|------|
| A1-瘟疫治疗 | +20 | 激光塔图纸×1 / 合金城墙图纸×1 | — |
| A2-石碑祭品 | +25 | 合金城墙图纸×1 | — |
| A3-残卷解谜 | +20 | 激光塔图纸×1 | — |
| A4-流浪商人 | +15 | 激光塔图纸×1 | 可能额外获得稀有资源 |

### 【类型 B】网页任务（Web Interact）— 工具使用

> **考察维度**：工具选择（OCR vs VLM vs 组合）、视觉理解、约束推理、最优决策、跨页面信息整合
> **状态**：🆕 新增

#### 3B.1 核心设计理念：L1 纯视觉模式

判题器内嵌浏览器引擎（Headless Chromium + Playwright），渲染完整的商店网页。Agent 收到的不是 HTML 源码，而是**网页截图（PNG）**。

```
判题器侧                              选手Agent侧
┌──────────────┐                    ┌─────────────────────┐
│ Playwright   │ ──截图PNG──→       │ 用什么读？           │
│ 渲染真实网页  │                    │ OCR？快但可能漏/错   │
│ 执行点击/滚动 │ ←──坐标(x,y)──    │ VLM？理解强但贵/慢   │
│ 返回新截图    │                    │ OCR+VLM交叉验证？    │
└──────────────┘                    └─────────────────────┘
```

**核心考察不是"能不能解析HTML"，而是 Agent 面对一张渲染好的网页截图时——**
- 选什么工具来读（OCR？VLM？两者都用？）
- 工具输出不一致时信谁？
- 首屏信息不够时，是否知道往下滚动？
- 折叠区能否识别并主动点击展开？
- 跨页面信息如何整合？

#### 3B.2 商品池（固定5件）

| # | 商品 | 效果 | 价格 | 标签 |
|---|------|------|:---:|------|
| 1 | 瞄准镜组件 | 防御塔攻击力 +20% | 45 | 攻击 |
| 2 | 纳米涂层 | 围墙防御力 +30% | 40 | 防御 |
| 3 | 脉冲发生器 | 防御塔射程 +1格 | 55 | 射程 |
| 4 | 能量核心 | 基地获得额外 800 HP | 80 | 防御 |
| 5 | 夜视目镜 | 英雄夜间视野 +2格 | 10 | 辅助 |

> **价格设计意图**：#1(45)+#3(55)=100 ≤ 预算100，两者**买得起**但被互斥规则拦住——互斥约束由规则驱动，不由预算驱动。

#### 3B.3 约束规则

```
① 每人 100 金币，超预算无法提交订单
② #1 瞄准镜 与 #3 脉冲发生器 互斥（二者占用防御塔同一接口槽位），不能同时勾选
③ 每人限购 2 件不同商品
④ 所有效果为永久生效
```

#### 3B.4 页面交互方式

所有题目类型共享同一交互方式——**简洁的勾选+购买**：

```
┌─────────────────────────────────────────────────────────┐
│     🌌 异界军火商 · 采购终端                            │
│                                                         │
│  📋 本次采购指令：[按题目类型变化，详见T1/T2/T5]         │
│  💰 余额：100 金币                                      │
│                                                         │
│  ┌───────────────────────────────────────────────────┐  │
│  │ #  商品        效果                 价格   勾选    │  │
│  │ 1  瞄准镜组件   攻击力+20%           45    [ ]    │  │
│  │ 2  纳米涂层    围墙防御力+30%        40    [ ]    │  │
│  │ 3  脉冲发生器   射程+1格             55    [ ]    │  │
│  │ 4  能量核心    基地+800HP            80    [ ]    │  │
│  │ 5  夜视目镜    英雄视野+2格          10    [ ]    │  │
│  └───────────────────────────────────────────────────┘  │
│                                                         │
│  ⚠️ 规则：互斥=1↔3 | 限购2件 | 预算100                   │
│                                                         │
│  [ 购买 ]                                               │
└─────────────────────────────────────────────────────────┘
```

**交互流程**：

```
Agent 勾选商品（点击复选框 [ ] → 变为 [✓]）
  → 点击 [购买] 按钮
    → 判题器验证：
        ✅ 通过 → 弹出成功提示 → 购买完成
        ❌ 失败 → 弹框显示具体原因：
           如："#1瞄准镜与#3脉冲发生器互斥，请重新选择。"
           或："总价超过100金币，请重新选择。"
           或："限购2件，请取消多余选择。"
         → Agent 需点击弹框上的 [确定]
         → 弹框关闭，回到购买页面，勾选状态保留
         → Agent 重新调整勾选 → 再次点击 [购买]
```

#### 3B.5 题目类型

##### T1：购物清单型（基础阅读+执行）

> 难度：★☆ | 考察：工具选择、基础视觉理解、照单执行

**指令行内容**：

```
📋 本次采购指令：
"前线指挥官要求购买【瞄准镜组件】和【纳米涂层】"
```

**验证**：45+40=85 ≤ 100，且 #1 与 #2 不互斥 → 正确勾选后即可通过。

**评分**：执行正确即满分，简单二元判断。

---

##### T2：约束寻优型（多约束下自主决策）

> 难度：★★☆ | 考察：约束理解、目标映射、权衡取舍、预算效率

**指令行内容**：

```
📋 本次采购指令：
"选择2件商品，最大化防御塔的攻击能力。
 注意：你的选择不能触发互斥规则。"
```

**Agent 决策**：

```
目标：最大化防御塔攻击能力
     → 5件商品中，与"攻击"直接相关的只有 #1 瞄准镜（攻击力+20%）
     → #3 脉冲发生器加的是射程，属于间接影响，不算攻击能力本身
     → #2/#4 是防御、#5 是视野，均与攻击无关

约束：限购 2 件，互斥 #1↔#3，预算 100

正确答案：
  #1 瞄准镜（45 金）——唯一直接提升攻击力的商品
  
其他组合的误判：
  - 选了 #3 脉冲发生器 → 未正确理解"攻击能力 ≠ 射程"
  - 选 #1+#3 → 违反互斥规则，提交被拒
  - 选了 #2 纳米涂层/#4 能量核心 → 混淆了"攻击"与"防御"
```

**评分**：选对 #1 即满分，选了其他不合理的则扣分。考察的是 Agent 对"攻击能力"这个概念的理解是否准确。

---

##### T5：信息拼图型（跨页面信息整合）

> 难度：★★★★ | 考察：多页面探索、信息拼凑、不遗漏关键信息、导航效率

**场景**：商品的效果、价格、购买规则、采购需求分散在多个页面。Agent 需要跨页面浏览，拼凑出完整决策信息。

主页只列出商品名称——效果和价格都是"详见规格页"。底部有链接入口指向规则页和情报页。Agent 必须主动点击链接跳转到子页面，获取分散在 5 个页面中的信息，最后回到主页提交购买。

---

### 【类型 C】推理任务（Intelligence）— 情报推演与双边博弈

> **考察维度**：因果链推理、噪声过滤、多因素交叉分析、谈判中利用信息差获取优势
> **状态**：🆕 新增

#### 3C.1 设计思路

协商博弈的核心不是"砍价技巧"，而是——**同一组情报，不同Agent推出不同结论，在贸易谈判中占据不同优势**。

晨间新闻系统（设计文档第十一章）每天推送多条情报。其中部分情报会真实影响游戏内的资源供给或消耗参数。Agent需要：
1. 从自然语言新闻中推理出"什么会变、何时生效、持续多久"
2. 据此调整采集、建造、贸易策略
3. 在贸易站与对方选手谈判时，利用信息差获取优势

**关键设计**：不引入金币/定价系统。情报直接影响游戏已有参数（采集产出、建筑消耗、贸易条件等），Agent推理的是**资源的供需变化**，谈判在已有的以物易物贸易站中进行。

#### 3C.2 情报的时效信息传达方式

情报中的时间信息**嵌在自然语言正文中**，Agent需要通过阅读理解提取，但不是猜谜——信息在文本里，只是非结构化：

| 传达方式 | 示例 | 适用场景 |
|---------|------|---------|
| NL描述（精确） | "约需1天渗透…清理需要2天" | 自然灾害/生态事件 |
| 引用历史先例 | "上个月铜矿区类似事件…间隔1天…恢复48小时" | 可类比的重复事件 |
| 明确公告 | "明日仅前60回合开放" | 系统级事件（贸易站维护） |
| 即时生效+永久 | "兽群已进化…建议重新评估火力" | 全局性永久变化 |

#### 3C.3 子类型

| 子类型 | 形式 | 对手 | 考察核心 |
|--------|------|------|---------|
| **C1-情报推演** | 晨间新闻 → 因果推理 → 调整采集/建造/贸易计划 | 市场（判题器影响参数） | 因果链推理、噪声过滤、时效提取 |
| **C2-双边贸易谈判** | C1推理结论 → 贸易站互报交换条件 → 谈判 | 对方选手 | 利用信息差、报价策略、对手建模 |
| **C3-囚徒困境** | 跨维度精灵提案 → 选择合作/背叛 | 对方选手 | 博弈论、信任评估、策略性沟通 |

#### 3C.4 情报示例（C1：情报推演）

每条情报包含：新闻正文 + 时效信息（嵌在文中） + 游戏实际影响（判题器侧）。

##### 示例1：兽群分泌物 → 铁矿石减产

```
📰 晨间新闻：

  "地质监测站报告：昨夜兽群迁徙路径经过北部铁矿区，
   地面检测到大量酸性分泌物残留。
   
   矿区领班表示：'目前开采仍在正常进行，但部分浅层矿面
   已经开始变色，矿工反映皮肤接触后有轻微灼烧感。'
   
   监测站工程师估计：酸性物质大约需要 1 天渗透到可采矿层
   深度，之后需要暂停开采进行化学中和处理，类似规模的
   污染通常需要 2 天左右才能清理完毕、恢复开采。"

时效信息（Agent需从文中提取）：
  生效时间：约 1 天后（明日）
  持续时间：约 2 天
  当前状态：今日仍正常开采

因果链（Agent需推理）：
  酸性分泌物 → 渗透矿层 → 腐蚀铁矿石 → 清理期间停产
  （1跳，化学-生产因果）

游戏实际影响：
  第N+1天至第N+2天：铁矿区采集产出 1.0 → 0.6

Agent合理对策（按深度分层）：
  Lv1：铁减产 → 今天多采铁囤着
  Lv2：铁减产 → 铁影响铁锭（围墙Lv3）、纯铁（防御塔Lv4）
        → 如果升级计划需要这些 → 囤铁；如果不需要 → 不囤
  Lv3：铁减产+同时有其他情报 → 交叉分析 → 重排全局优先级
```

##### 示例2：树皮甲虫爆发 → 木材减产

```
📰 晨间新闻：

  "护林员紧急报告：密林区域发现树皮甲虫灾害。
   害虫钻入树干内部蛀蚀，被感染的树木木质脆化、无法
   用于建筑加工。
   
   护林员称虫害大约在 3 天前首次被发现，扩散速度正在加快。
   根据以往虫害记录，从扩散加速到木材可用率明显下降，
   通常只有 1 天左右的窗口期，虫害高峰期会持续 2-3 天。"

时效信息（Agent需从文中提取）：
  生效时间：约 1 天后
  持续时间：2-3 天
  参考历史：护林员引用了"以往虫害记录"作为依据

因果链：
  甲虫钻入树皮 → 蛀蚀树干 → 木材结构破坏 → 可用的好木材变少
  （1跳，生物-生态因果）

合理性：
  树皮甲虫是真实世界中的森林主要害虫（北美/欧洲每年因此
  损失数百万公顷森林），虫蛀木材不能用于建筑是常识。

游戏实际影响：
  第N+1天至第N+3天：密林区域木材采集产出 1.0 → 0.5

Agent合理对策：
  - 木材是几乎所有建筑的基础材料，减产影响面广
  - 囤木材，或推迟非紧急建筑计划到虫害结束后
```

##### 示例3：火山地热异常 → 硫磺暂停开采

```
📰 晨间新闻：

  "山脉监测站报告：东部火山口地热活动近日持续增强，
   硫磺矿脉周边的地热喷口数量增加了 3 处，部分喷口温度
   已超出安全阈值。
   
   监测站已将预警等级提升为'黄色'，站长表示：'根据安全
   规程，黄色预警意味着未来 1-2 天内可能需要临时封闭
   硫磺开采区。上个月红色预警时封闭持续了整整 2 天，
   黄色预警的封闭时间通常也是这个标准。'"

时效信息（Agent需从文中提取）：
  生效时间：未来 1-2 天内（有不确定性）
  持续时间：约 2 天（引用上个月先例）
  当前状态：今天仍可开采

因果链：
  地热活动增强 → 安全规程要求封闭 → 硫磺无法采集
  （1跳，地质→安全规程。注意：封闭是因为预警机制，
   不是火山真的喷发了——Agent需区分"预防性封闭"和"实际灾害"。）

合理性：
  火山预警→预防性封锁是现实标准操作（参考樱岛火山、
  基拉韦厄火山周边的定期封锁）。黄色预警不是最高级，
  但安全规程是强制性的。

游戏实际影响：
  第N+1天（或N+2天）起，连续2天：硫磺采集产出降为 0
  （判题器在黄色预警的第2天早上根据随机概率决定是否触发封闭）

Agent合理对策：
  - 硫磺→火药→步枪塔(Lv3)必需品
  - 如果计划升级防御塔，趁今天多采硫磺
  - 有不确定性（1-2天内）→ 保守策略是今天立刻采；激进策略是赌第2天才封
```

##### 示例4：贸易站通道波动 → 明日交易窗口缩短

```
📰 晨间新闻：

  "跨空间贸易站公告：侦测到维度通道能量出现周期性波动。
   根据校准协议，明日贸易站仅能在白天前 60 回合维持
   稳定连接，之后通道将暂时关闭进行自动校准。
   校准预计在次日（后天）完成，届时恢复正常运营。
   今日贸易站不受影响。"

时效信息（Agent需从文中提取）：
  今日：正常
  明日：仅白天前 60 回合可用
  后天：恢复正常
  这是确定性公告（系统维护），无不确定性

因果链：
  通道波动 → 自动校准协议触发 → 交易窗口压缩
  （1跳，技术-时间约束）

博弈价值：
  - 双方都知道窗口短 → 明天谈判有时间压力
  - 如果对方急需你的独占资源（铜/硅）来完成芯片，
    他无法拖延——60回合内必须成交或放弃
  - 你可以在谈判中利用这个时间压力争取更好的交换比例

游戏实际影响：
  第N+1天仅在白天前 60 回合可进行贸易

Agent合理对策：
  - 浅层：明天早点去贸易站
  - 中层：对方也知道窗口短 → 如果对方更急→你有谈判优势
  - 深层：如果你的升级不依赖明天的交易→故意拖到窗口快关闭
         测试对方急不急 → 如果不急就正常交易
```

#### 3C.6 晨间新闻频道结构

C 和 D 共享晨间新闻系统，但信息归属不同频道——Agent 不需要猜测每条信息属于哪个任务，频道标签本身已经做了区分。

```
═══════════════════════════════════════════════════════════
  📰 第 N 天晨间新闻
═══════════════════════════════════════════════════════════

【兽潮预警】           ← 每天必读，结构固定，简短
【世界事件】           ← C1 情报（全为真），每天 1-2 条，影响游戏参数
【各地见闻】           ← G 碎片线索（全为真），每天 0-1 条，分散在 10 天中

═══════════════════════════════════════════════════════════
```

| 频道 | 频率 | 内容性质 | 对应任务 | Agent 处理方式 |
|------|:---:|------|:---:|------|
| 【兽潮预警】 | 每天 | 今晚兽潮种类+数量 | — | 评估防守压力，调整回防策略 |
| 【世界事件】 | 每天 1-2 条 | 客观情报，影响采集/建造/贸易参数 | C1 | 因果推理→立即调整策略 |
| 【各地见闻】 | 每天 0-1 条 | NPC 见闻碎片，指向隐藏宝藏 | G | 判断→存入长期记忆→跨天整合 |

**设计意图**：每个频道的职责边界明确。【世界事件】需要今天做决策，【各地见闻】需要今天记住改天用——处理策略完全不同，考察 Agent 的信息分类和长上下文管理。

### 【类型 D】长上下文任务（Fragment Quest）— 记忆整合

> **考察维度**：长上下文记忆管理、跨回合信息拼凑、碎片推理、时机判断
> **状态**：🆕 新增

#### 3D.1 设计思路

将一条完整信息拆成 5 个碎片，分散在 10 天晨间新闻的 **【各地见闻】** 频道中。每天 0-1 条，每条碎片单独看都像"NPC的日常闲聊"，但拼在一起才揭示一条完整线索。

**与 C1 情报推演的关键区别**：

| | C【世界事件】 | D【各地见闻】 |
|------|------------|------------|
| 信息来源 | 监测站/实验室/军方 | NPC口述（矿工/猎人/诗人/士兵） |
| 信息性质 | 客观事实，影响游戏参数 | 主观见闻，指向隐藏宝藏 |
| 单条独立？ | 是，每条独立可推理 | 否，单条无意义，拼起来才行 |
| 需要行动？ | 推理后调整采集/建造/贸易策略 | 集齐5条后推理，在特定时机到特定地点执行 |
| 真实性 | 100%真 | 100%真（不做噪声） |

**Agent 需要做的**：

1. **识别碎片**：在每天晨间新闻中，区分【世界事件】（C1情报，需立即推理行动）和【各地见闻】（碎片，需记忆暂存）。二者的频道标签不同，不需要猜测。

2. **跨天记忆**：将碎片存入长上下文。5条碎片分散在10天中，最长间隔达5天（第4天→第7天之间）。

3. **发现碎片齐全**：第10天收到最后一条后，Agent应意识到5条碎片已集齐，触发整合推理。

4. **整合推理**：
```
碎片1（Day 1）：双子峰之间有石门，只在特定日子显现
      ↓
碎片4（Day 8）：新月那晚，哨塔影子指向双子峰山口
      ↓  组合推理：石门显现的"特定日子"=新月之夜，
      │           入口位置=哨塔影子所指之处
      ↓
碎片7（Day 7）：三种祭品——大地、深海、天空
      ↓  物品映射：大地=矿石/铁锭、深海=纯硅、天空=火药
      ↓            （这些是游戏内已有的材料，Agent需要做映射）
      ↓
碎片4（Day 4）：石壁上有古代符号，只有月光照到才能看见
      ↓  组合推理：石壁上的符号可能是祭品放置顺序的提示
      ↓
碎片10（Day 10）：门在影落之处，钥在三人同行之时
      ↓  组合推理："影落之处"=哨塔影子指向的位置
      │               "三人同行"=需要3名英雄同时在入口处
```

5. **时机执行**：在新月之夜（第7晚，但第7天碎片才出现，Agent可能已经错过。需要在第10天整合后确认下个新月时间或等待游戏提示），派3名英雄到双子峰哨塔影子的落点，按正确顺序使用祭品。

**完整答案**：
```
地点：双子峰之间的山口，废弃哨塔在新月之夜的影子落点
时机：新月之夜（游戏内的特定夜晚）
条件：3名英雄同时在场
祭品：铁锭（大地）、纯硅（深海）、火药（天空），按石壁符号顺序使用
```

#### 3D.3 考察维度分层

| 层级 | 能力 | 体现 |
|:---:|------|------|
| Lv1 | 频道识别 | 能区分【各地见闻】和【世界事件】，知道碎片需要存下来 |
| Lv2 | 跨天记忆 | 10天内碎片不丢失，第10天集齐后触发整合 |
| Lv3 | 信息整合 | 将5条碎片拼成完整线索，正确推理出地点、时机、条件 |
| Lv4 | 物品映射 | "大地→铁锭、深海→纯硅、天空→火药"——将NL描述映射到游戏内已有物品 |
| Lv5 | 时机执行 | 在正确的时间派正确数量英雄到正确地点使用正确物品 |

## 四、任务调度与节奏设计

### 4.1 10天任务节奏总览

```
Day  Nights         Daytime Tasks                          Night
 1  小型兽×2-3    新闻: 长流程任务#1发布+推理任务开始        防守教学
                  【各地见闻】碎片#1
                  
 2  小型兽×4-5    网页任务可触发(T1/T2)                    基础防守
                  
 3  中型兽×1-2    C2双边谈判可触发                        围墙压力测试
                  
 4  中型兽×2-3    网页任务可触发(T5)                       火力考验
                  【各地见闻】碎片#2
                  
 5  大型兽×1      情报4(贸易窗口)发布                      Lv3塔刚需
                  
 6  大型兽×2      新闻: 长流程任务#2发布                    全面防守
                  C3囚徒困境可触发
                  情报5(中型兽进化)发布
                  
 7  大型兽×3      【各地见闻】碎片#3                       星月夜
                  
 8  大型兽×4-5    【各地见闻】碎片#4                       激光塔刚需
                  
 9  BOSS×1        最后冲刺: 完成未完成的任务              极限防守
                  
10  BOSS×2        【各地见闻】碎片#5                       最终审判
                  碎片整合 → 宝藏开启窗口
```

### 4.2 调度原则

1. **不堆砌**：每天最多触发 1-2 个新任务，保持节奏感
2. **渐进复杂**：前期任务简单（网页T1/T2），后期任务综合（网页T5、碎片整合）
3. **信息分层**：晨间新闻按频道区分（兽潮预警/世界事件/贸易站公告/各地见闻），Agent根据频道标签决定处理策略
4. **自由选择**：选手可以跳过任何任务（但有代价——拿不到强力奖励）
5. **重复可玩**：每局从任务池中随机抽取，不同对局体验不同

### 4.3 任务池与随机抽取

```
每局比赛任务配置：
  - 长流程任务（A）：固定 2 个（第1天+第6天各1，从4种类型中抽取）
  - 网页任务（B）：1-2 个（T1/T2/T5，难度递进）
  - 推理任务（C）：C1 每日晨间新闻【世界事件】自动推送，C2/C3 随机触发
  - 碎片任务：1 组（5条碎片分散在10天【各地见闻】中）
```

## 七、与现有系统的兼容

### 7.1 对已有游戏系统的影响

| 系统 | 影响 | 说明 |
|------|:---:|------|
| 英雄系统 | 无变化 | 开拓者仍是唯一任务执行者 |
| 建筑系统 | 无变化 | 奖励作用于已有建筑属性 |
| 昼夜系统 | 无变化 | 任务仅在白天可触发 |
| 积分系统 | 微调 | 新增任务积分项（见5.2） |
| 资源系统 | 无变化 | 新增奖励资源走已有仓库系统 |
| 贸易系统 | 无变化 | F类型任务在贸易站旁触发，不修改贸易规则 |

### 7.2 选手感知

对选手而言，所有新增任务都通过"开拓者探索"自然触发，感知上是**"游戏内容更丰富了"**而非"考核维度增加了"。选手只需要做好一件事——**写好Agent让它能处理各种类型的探索结果**——好的Agent自然会在各维度上得分高。
