## 三、整改优先级建议

### 可立即整改（零或极低代码量）

| 优先级 | 组件 | 说明 |
|-------|------|------|
| P0 | axum-extra | 零引用，直接删除 |
| P0 | tokio-stream | 零引用，直接删除 |
| P0 | anyhow | 零引用（项目已禁止使用），直接删除 |

### 低难度（1-2个文件，API 调用简单）

| 优先级 | 组件 | 说明 |
|-------|------|------|
| P1 | tower-http | 1文件，CORS middleware 可手写或换内置方案 |
| P1 | serde_yaml | 2文件，换 serde_yml 等替代 |
| P1 | json5 | 1文件，JSON5格式特殊需谨慎 |
| P1 | ssdp-client | 1文件 |
| P1 | pnet/dhcproto/nusb | 各1文件，可选 feature，关闭即移除 |
| P1 | tokio-tungstenite | 仅 Matter 模块使用 |

### 中难度（涉及核心逻辑但范围可控）

| 优先级 | 组件 | 说明 |
|-------|------|------|
| P2 | axum | 8文件，HTTP 框架迁移量较大 |
| P2 | futures-util | 7文件，但 tokio-stream 可覆盖部分功能 |
| P2 | rusqlite | 2文件，需替换 SQLite binding |
| P2 | tracing-subscriber | 1文件，换日志后端 |
| P2 | voluptuous (Python) | HA 生态广泛使用，API 表面可控 |

### 高难度（遍布全仓，替换代价大）

| 优先级 | 组件 | 说明 |
|-------|------|------|
| P3 | tracing | 34文件遍布全仓，Rust 生态事实标准 |
| P3 | pyo3 | 18文件，整个 Python bridge 的基石 |
| P3 | aiohttp (Python) | HA 生态深度依赖，shim 中有专门适配层 |

### 不需要整改

tokio, wasmtime, serde, serde_json, thiserror, anyhow(零使用), async-trait, chrono, libc, uuid, base64, libloading, attrs, jinja2, pillow, cryptography, certifi, charset-normalizer, PyJWT
